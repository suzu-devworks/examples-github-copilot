# Maintainability

## Purpose

Keep README files focused on stable, repository-level information that remains useful as the repository evolves.

## Prefer

Prefer information that changes infrequently, such as:

- Repository purpose.
- Repository-wide conventions.
- Shared development environment.
- Long-term design decisions.

Keep documentation easy to review and maintain.

- Keep paragraphs reasonably short.
- Avoid unnecessary repetition.
- Prefer simple, direct wording.
- Group related information together.

## Avoid

Avoid information that is likely to become outdated, including:

- Project-specific setup.
- Build or execution steps.
- Project-specific configuration.
- Version-specific instructions.
- Frequently changing feature lists.

Avoid duplicating documentation that already exists elsewhere.

Avoid documenting information that belongs in an individual project's own README.

Avoid including guidance from this skill unless it describes an actual characteristic of the repository.

## Goal

A repository README should remain useful even as projects are added, updated, reorganized, or removed.
